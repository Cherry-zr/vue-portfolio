# 限制与推断

- 参考视频分辨率约为手机竖屏，只覆盖首页频道、健康页、课堂、采购清单和侧边菜单；搜索、分类、详情、话题和用户页主要依据接口数据与项目报告截图合理设计。
- 视频中的减重、课堂、AI 和采购清单没有对应公开接口，因此不实现其业务页面。
- 原生 App 的字体、精确动效时长、系统状态栏和 WebView 行为没有验证。
- 浏览器实际验收覆盖了桌面视口中的 430px 移动端容器；页面同时使用 `width: 100%` 和 430px 上限适配窄屏，但本次浏览器连接不支持把外层视口精确改为 390px。
- 远程接口、图片与返回字段可能变化；页面通过适配器和错误状态降低影响。
- 2026-07-19 验证时，上游允许 `https://cherry-zr.github.io` 跨域读取；生产环境直接使用 HTTPS 上游接口，并保留错误态应对 CORS 策略变化。
- 登录缺失意味着“我的”、真实收藏写入、关注和发布无法完成，只提供明确禁用状态。
- 2026-07-18 验收时，菜谱详情和用户资料接口返回签名验证失败，食材详情与营养分析返回空响应；相关内容、评论、学做、相关笔记、用户作品和用户日历仍可读取。
